Learnings

What this course project strengthened (design + testing mindset)

Key Takeaways

Design artifacts improve testability

Decomposition + sequence diagrams reduce ambiguity and make it easier to derive test cases from flows.

Test planning needs clear entry/exit

A test plan becomes actionable when it defines scope, environments, responsibilities, and exit criteria.

Performance tests should be repeatable

JMeter plans with explicit users/ramp/duration make results comparable and useful for iteration.

NFR coverage is a checklist, not an afterthought

Security, usability, compatibility, and database testing each need concrete test assets and evidence.

Next Improvements (If Iterating)

  • Add explicit pass/fail criteria per test case and link to defects (traceability)
  • Standardize naming for test packs and group them by priority/risk
  • Export JMeter results (JTL/HTML dashboard) and attach as evidence
  • Add a minimal requirements traceability table (Use Case → Tests)
