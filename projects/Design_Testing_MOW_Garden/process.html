Business Analysis / QA Process

How the MOW Garden project was structured from design to testing deliverables

Timeline & Phases

01

Scope & Use Cases

Focus: define core modules and user goals

  • Identify main flows (catalog, cart, checkout, inventory, access)
  • Capture use cases in a structured template
  • Agree on assumptions and boundaries (course project constraints)
02

Design Views

Focus: make flows consistent and testable

  • Decomposition diagram to clarify responsibilities
  • Sequence diagrams per module (cart, payment, catalog, inventory)
  • Access control and communication views
03

Test Planning & Test Design

Focus: define strategy + concrete test assets

  • Test plan: approach, environments, entry/exit criteria
  • Functional test cases + review checklist
  • NFR coverage: usability, compatibility, security, database testing
04

Performance Testing (JMeter)

Focus: run repeatable load tests with clear parameters

  • Homepage load test plan (50 users, 300s, ramp 30s)
  • API performance plan (auth + browse + checkout endpoints)
  • Collect results via Summary/Aggregate reports
05

Reporting & Improvements

Focus: turn findings into actionable improvements

  • Defect logging template for consistent reporting
  • Use results to refine test coverage and critical flows
  • Document learnings for iteration
