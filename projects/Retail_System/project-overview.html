Retail Operations Backoffice

Turning a course assignment into a recruiter-ready BA case study

Problem Statement

Many small and mid-sized retail stores run daily operations using spreadsheets + informal processes. This leads to stock variance, inconsistent pricing/promotions, weak audit trails, and slow reporting. The goal of this project is to define a clear, end-to-end backoffice workflow that supports multi-branch operations across sales, inventory, purchasing, transfers, and finance.

Objectives

  • Standardize processes across branches with clear use cases and flows.
  • Increase traceability with consistent actors, permissions, and audit events.
  • Reduce operational errors (stock mismatch, incorrect promotions, cashbook issues).
  • Enable faster onboarding by documenting how the system should behave.

Scope (MVP Modules)

Sales (Bán hàng)

Checkout, invoices, customer payments, and sales lifecycle.

Inventory (Kiểm kho)

Stocktake, variance handling, and reconciliation.

Purchasing (Nhập hàng)

Receiving goods, supplier invoices, and returns to supplier.

Transfers (Chuyển hàng)

Inter-branch transfers and transfer receiving workflows.

Promotions (Khuyến mại)

Promotion setup, validation rules, and application logic.

Reporting (Thống kê)

Summary dashboards for management decisions.

Full module diagram set is available on the Deliverables page.

What Exists Today (Course Output)

  • Use Case diagrams per module (UC).
  • Sequence diagrams for key flows (sequence).
  • Activity diagrams for process logic (module workflow).
  • UI screen design (where available / to be expanded).

Recruiter-Grade Upgrades (Planned)

  • PRD-lite: goals, scope, assumptions, constraints, NFRs, and edge cases.
  • Requirements structure: user stories + acceptance criteria aligned to UC/sequence.
  • Traceability: UC → requirements → test cases → defects (simple but credible).
  • UAT pack: test cases template + summary report for 1–2 core scenarios.
